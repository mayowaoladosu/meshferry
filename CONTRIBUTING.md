# Contributing

Thanks for contributing to MeshFerry.

## Development loop

1. Install dependencies with `npm install`.
2. Run `npm run typecheck`.
3. Run `npm run build`.
4. Test changes locally against a simple HTTP app before opening a pull request.

## Pull request guidelines

- Keep changes focused.
- Update the README when behavior changes.
- Prefer small, well-explained pull requests over broad rewrites.
- Include reproduction steps for bug fixes.

## Scope guidance

Before adding major features, open an issue and tie the proposal back to the roadmap. The project should keep a narrow tunnel-first core while the APIs are still settling.
